# Fomalhaut 风格首页与侧栏改造设计

- Date: 2026-04-09
- Project: `pasule-blog`
- Reference: `hexo-theme-Fomalhaut` 本地仓库 + `https://www.fomal.cc/posts/76815704.html`
- Design status: Drafted after interactive brainstorming, pending user review

## 1. 目标

在保留当前站点核心世界观的前提下，把博客首页与侧栏改造成更接近 Fomalhaut / fomal.cc 的内容站入口形态。

本轮改造的重点不是整站 1:1 复刻，而是：

- 明显借鉴 Fomalhaut 的首页首屏结构、侧栏挂件密度与内容站气质
- 保留当前站点的背景图和 Live2D
- 保留现有主要页面入口与博客内容体系
- 让首页第一眼更有“高完成度的个人技术站”感觉
- 为后续再接更多挂件和脚本留出空间

## 2. 已确认边界

### 2.1 保留项

以下内容在第一轮中明确保留：

- 现有背景图体系
- 现有 Live2D
- 现有主要页面入口：首页 / 归档 / 标签 / 分类 / 说说 / 相册 / 项目 / 留言板 / 友链 / 关于
- 当前基于 Butterfly 的主题底座

### 2.2 可调整项

以下内容允许自由重构或重写：

- 顶部导航结构与层级
- 首页首屏结构
- 首页精选区布局
- 侧栏挂件内容与顺序
- 首页与侧栏交互脚本
- 局部组件风格
- 轻量开源依赖的引入方式

## 3. 参考站可借鉴内容

Fomalhaut 的价值不只是配色，而是“综合内容站入口”的组织方式。对当前站点最值得借鉴的部分有：

- 大轮播主导的首页首屏
- 分组式导航
- 高密度但有层次的侧栏卡片
- 首页轮播下的精选文章层
- 系列入口卡 / 专题传送门
- 阅读进度与导航标题切换等“完成度脚本”
- 显著的公告、欢迎和状态信息面板

不建议第一轮直接照搬的部分：

- IP 地理位置欢迎语
- 大量节庆特效和常驻粒子特效
- 整套 Vue2 + Element UI 弹窗面板体系
- 音乐播放器常驻首页
- 过多的外部 CDN 杂项脚本

## 4. 导航设计

### 4.1 总体策略

导航采用“半分组式”而不是完全扁平，也不是完全照搬 Fomalhaut 的栏目命名。

设计原则：

- 保留当前页面入口
- 重组为更像内容站的导航层级
- 桌面端突出分组感
- 移动端优先保证稳定可用

### 4.2 推荐导航分组

建议分成如下组：

- `首页`
- `文章`
  - 归档
  - 标签
  - 分类
- `生活`
  - 说说
  - 相册
- `站点`
  - 项目
  - 留言板
  - 友链
- `关于`

这个结构比当前更像 Fomalhaut 的站点型导航，但不强行移植它的全部栏目体系。

## 5. 首页首屏设计

### 5.1 首屏定位

首页采用 `大轮播主导` 的首屏结构。

这意味着：

- 首页第一视觉重心不再是普通文章流
- 首屏主角是轮播
- 文章流退到首屏后方
- 轮播承担“定义站点气质”的角色

### 5.2 轮播内容来源

轮播采用 `手动精选 3 到 5 篇文章` 的方式，不做自动抓取。

原因：

- 手动精选更像 Fomalhaut 的“首页编排”
- 更容易控制首页气质
- 避免自动抓取导致质量不稳定
- 与当前 `content-map.yml` 的中心化内容映射方向一致

### 5.3 轮播卡信息

每张轮播卡建议展示：

- 封面图
- 标题
- 一句摘要
- 分类或系列标签
- 进入文章按钮

轮播视觉气质应更贴近 Fomalhaut：

- 更强的圆角卡片感
- 更明显的封面主导
- 更清晰的内容重心
- 但不替换当前背景图系统

## 6. 首页主区结构

首页首屏轮播之后，不应立刻进入普通文章列表，而应先进入“精选内容层”。

### 6.1 精选文章区

功能：

- 承接轮播之外的重点文章
- 让用户在首屏之后仍处于“被引导”的状态

建议内容：

- Java 并发
- MySQL 索引优化
- 微服务架构
- Git 工作流
- 前端性能优化

### 6.2 系列入口区

这部分直接借 Fomalhaut 的“专题传送门”思路。

建议做 4 张系列卡：

- Java / 后端成长线
- 数据库性能线
- 前端工程线
- 工程协作线

卡片应该是系列入口，而不是单篇文章卡。

### 6.3 项目速览区

首页上半部分要有项目入口，避免项目页继续成为深层页面。

这部分应：

- 与精选文章区保持同一视觉语言
- 让“博客 + 项目站”定位更明确
- 承接现有项目页改造成果

## 7. 侧栏挂件设计

### 7.1 侧栏总策略

侧栏要更像 Fomalhaut 的“信息面板”，但第一轮不追求最大密度，而追求“高价值密度”。

### 7.2 第一轮优先挂件

建议优先保留或增强以下卡片：

- 公告 / 欢迎卡
- 快速跳转卡
- 站点信息卡
- 最近文章卡
- 系列入口卡

### 7.3 侧栏风格目标

侧栏应该呈现：

- 更统一的半透明卡片体系
- 更明显的信息分层
- 更像面板而不是普通博客附属栏
- 与首页主区保持同一视觉语言

## 8. 个人化脚本与交互

### 8.1 第一轮优先脚本

第一轮优先接入这些“高价值、低副作用”的交互：

- 阅读进度百分比
  - 用于替代或增强回顶按钮
- 导航标题切换
  - 下滚时显示当前页面标题
  - 上滚时恢复站点标题
- 首页公告 / 欢迎区增强
  - 先做静态增强版
- 轻量入场动画
  - 轮播区、精选卡片、侧栏卡片轻量动效

### 8.2 第二轮再考虑的脚本

这些不纳入第一轮主目标：

- IP 地理位置欢迎语
- 节庆灯笼 / 雪花 / 元宝 / 烟花类强装饰特效
- WinBox 弹窗式美化面板
- 音乐播放器常驻首页

## 9. 昼夜切换需求

用户明确要求首页与站点体验里加入更像 Fomalhaut 的昼夜切换体验。

### 9.1 本轮目标

本轮要实现的不是“单纯有个 darkmode 按钮”，而是：

- 昼夜切换入口明显可见
- 切换体验足够像一个被认真设计过的功能
- 昼夜模式切换后首页壳、侧栏卡、轮播和内容区视觉都协同变化

### 9.2 实现建议

优先复用 Butterfly 现有 darkmode 能力，而不是额外引入一套重型 dark mode 插件。

原因：

- 现有主题本身已经支持昼夜切换
- 当前站点已有背景与主题变量体系
- 重型额外插件会增加维护成本和冲突风险

因此推荐做法是：

- 保留现有 darkmode 基础能力
- 重做昼夜切换入口的样式与位置
- 为首页和侧栏补齐昼夜模式下的视觉变量
- 如现有逻辑不足，再增加轻量增强脚本，而不引入大型框架依赖

### 9.3 设计效果目标

昼夜切换应达到以下感受：

- 不是普通开关，而是站点体验的一部分
- 白天模式更像轻盈玻璃卡内容站
- 夜间模式更像霓虹感、深色玻璃卡内容站
- 切换后背景、挂件、轮播和信息卡整体协调

## 10. 开源依赖策略

### 10.1 第一轮推荐引入

推荐优先考虑的依赖：

- `hexo-butterfly-swiper`
  - 用于首页大轮播
- `hexo-butterfly-tag-plugins-plus`
  - 用于后续文章外挂标签与内容块增强
- `hexo-butterfly-wowjs`
  - 用于轻量滚动入场动画
- 当前主题已有的灯箱 / lazyload 能力优先复用

### 10.2 第一轮不推荐引入

第一轮不建议直接引入：

- Vue 2
- Element UI
- WinBox
- 大量节庆脚本
- 过多外部 CDN 脚本杂项

### 10.3 原则

依赖策略选择 `中等接受`：

- 可接受常见开源库
- 但要避免又老又重又杂的依赖堆积
- 能自己写的小型交互尽量自己写

## 11. 第一轮实施优先级

推荐顺序：

1. 首页轮播与精选区
2. 半分组导航
3. 侧栏挂件增强
4. 阅读进度与导航标题切换
5. 昼夜切换入口样式与模式视觉协同

这个顺序可以让首页最快接近 Fomalhaut 的第一印象。

## 12. 风险与控制

### 风险 1：风格像了，但内容被压住

控制方式：

- 首屏轮播之后尽快进入精选文章与系列入口
- 不让首页只有视觉没有内容路线

### 风险 2：移动端菜单和轮播失稳

控制方式：

- 分组导航先优先保证 PC 端效果
- 移动端以可用性优先

### 风险 3：挂件太多导致首页失衡

控制方式：

- 第一轮只上高价值挂件
- 不把所有 Fomalhaut 小组件一次性搬满

### 风险 4：昼夜切换只是开关，没形成体验

控制方式：

- 把昼夜模式当成首页与侧栏视觉系统的一部分来设计
- 而不是仅仅保留默认按钮

## 13. 第一轮验收标准

第一轮完成后，应满足以下结果：

- 首页打开时第一眼明显更像 Fomalhaut 风格站点
- 首页大轮播成为主视觉焦点
- 导航层级比现在更像内容站
- 侧栏挂件更有信息面板感
- 背景图与 Live2D 仍保留
- 昼夜切换体验显著增强
- 首页整体更完整，但不会因为特效过多而杂乱

## 14. 总结

这次改造不是整站复刻 Fomalhaut，而是做一次“气质迁移”：

- 借首页大轮播与内容站入口感
- 借高密度但分层清晰的侧栏面板
- 借阅读进度与导航标题切换等完成度脚本
- 增强昼夜切换体验
- 保留当前站点自己的背景图与 Live2D 世界观

如果用户审阅通过，下一步应写成实施计划，再进入真实改造。
