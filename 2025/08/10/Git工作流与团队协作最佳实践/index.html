🔄 主流Git工作流

1. Git Flow

适用于发布周期固定的项目。

分支说明：

  • main: 生产环境代码
  • develop: 开发主分支
  • feature/*: 功能分支
  • release/*: 发布分支
  • hotfix/*: 紧急修复分支

2. GitHub Flow

适用于持续部署的项目，简单高效。

流程：

  1. 从main创建分支
  2. 开发并提交
  3. 创建Pull Request
  4. 代码审查
  5. 合并到main并部署

3. Trunk-Based Development

适用于高度自动化测试的团队。

特点：

  • 所有开发者直接向main提交
  • 分支生命周期很短（几小时到1天）
  • 依赖Feature Flag控制功能发布

🔀 合并策略

1. Merge Commit

优点：保留完整历史，可追溯
缺点：历史记录较多

2. Squash Merge

优点：历史简洁
缺点：丢失详细提交记录

3. Rebase

优点：线性历史，整洁
缺点：改写历史，需谨慎使用

使用VS Code解决冲突

VS Code提供了直观的冲突解决界面：

  • Accept Current Change: 保留当前分支修改
  • Accept Incoming Change: 使用合并进来的修改
  • Accept Both Changes: 保留两者
  • Compare Changes: 对比查看差异
